# Moonlight - Project Rules

## Golden Rule
Damla is the CEO. Listen to her, execute what she says. Don't stop the flow to ask for approval on every little thing. She gives the direction, you build it. If something is genuinely unclear, ask. Otherwise just do it.

## Project
Moonlight is a manifest (manifestation) app. Main screen shows moon phase illustration + astrological reminders/guidance (retrogrades, transits, moon phase advice). Other features (horary, astrology, tarot, dictionary) are accessible via swipe menu.

## Design Style
Sunny app style: hand-drawn Procreate illustrations, doodle typography, friendly personality. Each moon phase gets its own unique illustration. Think cozy, mystical, cute.

## Development Rules
- Build like a building: foundation first, bricks, plaster, decoration. Never jump around.
- Break everything into smallest pieces.
- Commit messages: lowercase english, no emojis, no dashes, comma separated.
- NEVER add Co-Authored-By or any co-author tag to commits.
